Preparación para la pregunta 21:

Para la pregunta 21, que vale 10 puntos, te recomendamos estudiar las diapositivas de la semana 04 donde se abordan las siguientes arquitecturas de sistemas distribuidos:

  • Cliente-Servidor: Separación clara de roles, escalabilidad centralizada.
  • Serverless: Escalado automático, pago por uso, reducción de la gestión de infraestructura.
  • P2P (Peer-to-Peer): Descentralización, resistencia a fallos, compartición de recursos.
  • Microservicios: Modularidad, despliegue independiente, escalabilidad granular.

Además, es crucial entender la importancia de los protocolos de comunicación en sistemas distribuidos, como HTTP entre otros.

Prepárate para diagramar un sistema distribuido complejo. Por ejemplo, considera la arquitectura de Yape, una aplicación de pagos móviles:

  • Frontend móvil:
    • Aplicación nativa para iOS (Swift) y Android (Kotlin)
    • Interfaz de usuario intuitiva para transferencias, pagos y gestión de cuentas
    • Implementación de seguridad biométrica (huella dactilar, reconocimiento facial)
  • Backend con microservicios:
    • Servicio de autenticación: gestión de tokens JWT, integración con OAuth 2.0
    • Servicio de transacciones: procesamiento en tiempo real, consistencia eventual
    • Servicio de notificaciones: push notifications, SMS, correos electrónicos
    • Servicio de gestión de usuarios: perfiles, preferencias, historial
  • Base de datos distribuida:
    • Uso de Apache Cassandra para alta disponibilidad y escalabilidad horizontal
    • Replicación multi-región para reducir la latencia y mejorar la resiliencia
  • Integración con sistemas bancarios externos:
    • APIs RESTful para comunicación con bancos asociados
    • Implementación de protocolos de seguridad bancaria (ej. 3-D Secure)
  • Capa de seguridad:
    • Encriptación end-to-end para datos sensibles
    • Implementación de firewall de aplicaciones web (WAF)
    • Sistema de detección y prevención de intrusiones (IDS/IPS)
  • Sistema de caché distribuido:
    • Uso de Redis para almacenamiento en caché de datos frecuentemente accedidos
    • Implementación de políticas de expiración y actualización de caché
  • Servicios de análisis de datos en tiempo real:
    • Uso de Apache Kafka para procesamiento de streams de datos
    • Implementación de Apache Spark para análisis de big data y machine learning

Practica dibujando estos componentes y cómo interactúan entre sí. Por ejemplo:

  • Muestra cómo el frontend se comunica con los microservicios a través de una API Gateway
  • Ilustra cómo los microservicios interactúan entre sí y con la base de datos distribuida
  • Representa el flujo de datos desde una transacción iniciada en el móvil hasta su procesamiento y almacenamiento
  • Dibuja cómo el sistema de caché distribuido se integra para mejorar el rendimiento de las consultas frecuentes
  • Visualiza cómo los servicios de análisis de datos en tiempo real procesan la información para generar insights

Recuerda incluir elementos de seguridad, como firewalls y capas de encriptación, para demostrar cómo se protege el sistema distribuido en múltiples puntos.
